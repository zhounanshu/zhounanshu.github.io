RankScores，难度等级:⭐️⭐️⭐️

 Write a SQL query to rank scores. If there is a tie between two scores, both should have the same ranking. Note that after a tie, the next ranking number should be the next consecutive integer value. In other words, there should be no “holes” between ranks.

+----+-------+
| Id | Score |
+----+-------+
| 1  | 3.50  |
| 2  | 3.65  |
| 3  | 4.00  |
| 4  | 3.85  |
| 5  | 4.00  |
| 6  | 3.65  |
+----+-------+

For example, given the above Scores table, your query should generate the following report (order by highest score):

+-------+------+
| Score | Rank |
+-------+------+
| 4.00  | 1    |
| 4.00  | 1    |
| 3.85  | 2    |
| 3.65  | 3    |
| 3.65  | 3    |
| 3.50  | 4    |
+-------+------+

题目意思：

编写SQL对得分表中的得分排名，允许存在并列的排名，但名次之间是连续的不允许有空缺。
例如：当两名选手得分相同且均为最高得分，则两者并列第一。接着对分数为第二高的选手进行排名，名词为第二名，以此类推。

解决方案：

思路： 对所有的记录进行唯一排名，对某一位选手的得分统计其得分在唯一排名表中的位置确定其名次

select s.Score, (select count(*) from (select distinct(Score) from \
Scores order by Score desc) as r where r.Score >= s.Score) as Rank from Scores as s order by Score desc;

Rising Temperature, 难度等级⭐️

Given a Weather table, write a SQL query to find all dates’ Ids with higher temperature compared to its previous (yesterday’s) dates.

# 示例表格
+---------+------------+------------------+
| Id(INT) | Date(DATE) | Temperature(INT) |
+---------+------------+------------------+
|       1 | 2015-01-01 |               10 |
|       2 | 2015-01-02 |               25 |
|       3 | 2015-01-03 |               20 |
|       4 | 2015-01-04 |               30 |
+---------+------------+------------------+

题目意思：

编写SQL找出Weather表中当天温度比前一天温度高的记录的Id

解决方案：

思路：查询前一天的温度，将当天的温度和前天的温度进行对比

select Id from Weather as cur where Temperature > \
(select Temperature from Weather where date_add(Date, interval 1 day) = cur.Date );

或者
SELECT wt1.Id
FROM Weather wt1, Weather wt2
WHERE wt1.Temperature > wt2.Temperature AND
      TO_DAYS(wt1.DATE)-TO_DAYS(wt2.DATE)=1;

Customers Who Never Order 难度等级⭐️

Suppose that a website contains two tables, the Customers table and the Orders table. Write a SQL query to find all customers who never order anything.

# Customers表
+----+-------+
| Id | Name  |
+----+-------+
| 1  | Joe   |
| 2  | Henry |
| 3  | Sam   |
| 4  | Max   |
+----+-------+

Table: Orders

# Orders表
+----+------------+
| Id | CustomerId |
+----+------------+
| 1  | 3          |
| 2  | 1          |
+----+------------+

题目意思：

编写SQL找出没有预定商品的客户

解决方案：
将客户表和订单表合并，找出CustomerId为null的值

select Name as Customers
from (select Customers.Id, Name, CustomerId from Customers left join Orders on Customers.Id = CustomerId) as s
where s.CustomerId is null;

Duplicate Emails, 难度系数：⭐️

Write a SQL query to find all duplicate emails in a table named Person.

# Person表
+----+---------+
| Id | Email   |
+----+---------+
| 1  | a@b.com |
| 2  | c@d.com |
| 3  | a@b.com |
+----+---------+

题目意思：

编写SQL找出Person表中邮件重复的记录

解决方案

通过group by找到每条记录重复的条数，选择重复条数大于1的记录

select distinct(Email) from (select Email, count(*) as counts from
Person group by Email) as t where counts>= 2;

Delete Duplicates Emals , 难度系数：⭐️

Write a SQL query to delete all duplicate email entries in a table named Person, keeping only unique emails based on its smallest Id.

# Id is the primary key column for this table.
+----+------------------+
| Id | Email            |
+----+------------------+
| 1  | john@example.com |
| 2  | bob@example.com  |
| 3  | john@example.com |
+----+------------------+

题目意思 ：

编写SQL删除表中重复的字段，同时保留唯一Id较小的字段

解决方案：
思路：采用inner join, 对表自身进行内连接，然后删除符合条件的记录

Delete p1 from Person p1 inner join Person p2
where p1.Email = p2.Email and p1.Id > p2.Id;
